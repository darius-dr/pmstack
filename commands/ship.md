# /pm:ship

You are applying an approved pmstack handoff plan to the site repo. Equivalent of gstack's `/ship` for marketing changes.

## Procedure

1. **Resolve the plan file**:
   - If `$ARGUMENTS` is a path, use it.
   - Otherwise, find the most recent `.pmstack/diff/*-plan.md` by mtime.
   - If none exist, tell the user to run `/pm:handoff` first and stop.

2. **Confirm with the user before editing any files**. Print the plan summary (file list + change types). Wait for explicit "go" or "apply" before proceeding.

3. **Re-run the trust audit one more time** by reading the most recent `.pmstack/audits/*.md`. If Critical > 0 since the plan was written, refuse to ship and explain why.

4. **Apply each change in the plan** using Edit (preferred) or Write (only for net-new files):
   - For each file in the plan, apply the documented before/after diff exactly.
   - If the "before" string doesn't match the current file content (drift), STOP and report which file drifted. Do not partial-apply.

5. **Run repository sanity checks** (if available):
   - `npm run lint` — if it fails, revert the changes and report.
   - `npm run build` — if it fails, revert and report (with the failing error).
   - `npm test` — if test files exist, run them. Failures: revert and report.

6. **Stage the changes for the user's review** but do NOT commit:
   - `git add -p` is preferred for human review.
   - Print the suggested commit message from the plan.
   - Print a one-line summary: "{N} files changed, {M} insertions, {K} deletions. Ready for your commit."

7. **Update `.pmstack/diff/{plan}-plan.md`** with a "Shipped at: {timestamp}" footer line.

## Hard rules

- Never commit, push, or open a PR. The user does that.
- Never apply a plan that hasn't been generated by `/pm:handoff` (no ad-hoc edits via this command).
- Never edit a file outside the plan, even if you notice an opportunity.
- On any error, revert all changes made in this run and leave the working tree clean.
